# テストチェックリスト

Quoridor の品質確認を、手動テストで見るべき体験面と、自動テストで継続的に守るべきロジック面に分けて整理する。

## 目的

- ルール、ターン進行、スキル、CPU、Unity 表示の退行を早期に検知する。
- 手動確認に依存しやすい観点と、自動化して毎回確認すべき観点を分離する。
- 現在の EditMode テストで最低限不足している領域を明確にし、優先順位を付けて追加する。

## 現在自動テストで確認済みの主な領域

- `PlayerState` のターン開始処理、ステータス期限、ステータス削除イベント、スキルのクールダウンや使用回数、ディープコピー。
- 壁配置パターンの不変条件と、既存壁を対象にした壁除去ルール・効果。
- 探索用状態 `SearchState` の初期化、手の適用・巻き戻し、CPU コマンド変換。
- `Pathfinder` と `SearchProfiler` の基礎的な計測連携。
- スキルボタンの表示スロット変換と共通 `Guard` の引数検証。

## 手動テストすべき観点

Unity Editor または実機ビルドで、見た目、入力感、演出、アセット接続、非決定的な CPU 体験を確認する。

### 起動・セッション

- [ ] `Standalone` シーンを開いて Play し、例外なしで初期盤面、手番表示、スキル UI、ステータス UI が表示される。
- [ ] 新規対戦開始後、1P/2P の初期位置、壁数、スキル、CPU 設定がプリセットどおりに反映される。
- [ ] リセット操作で盤面、履歴、手番、選択状態、演出状態が初期化される。
- [ ] 勝敗確定後に追加操作が受け付けられず、結果表示や操作可能状態が破綻しない。

### 盤面入力・基本ルール

- [ ] 移動可能セルのクリック/タップで駒が移動し、手番が交代する。
- [ ] 盤外、壁セル、到達不能セル、相手駒と矛盾するセルへの入力が拒否され、UI が拒否理由を分かりやすく示す。
- [ ] 壁配置の向き、長さ、交差、重複、隣接、残数不足、経路封鎖禁止が実プレイ上で期待どおりに働く。
- [ ] マウス操作、タッチ相当操作、連打、ドラッグ外し、ボタン外クリックで選択状態が破綻しない。

### スキル・ステータス

- [ ] 通常移動、壁配置、壁除去などの組み込みスキルが、対象選択から効果反映まで一貫して動作する。
- [ ] クールダウン中、使用回数切れ、対象種別不一致、発動不可ステータス中のスキルが UI 上も実行上も無効化される。
- [ ] 睡眠、麻痺などのステータス付与、残りターン表示、解除、再付与時の表示更新が正しい。
- [ ] 複数ステータス・同一ステータスの重複時に、アイコン数、説明、解除タイミングが分かりやすい。

### CPU・探索

- [ ] CPU 対戦で CPU が合法手のみを選び、一定時間内に手を返す。
- [ ] 探索時間制限、評価プリセット、CPU/人間の組み合わせを変えてもフリーズしない。
- [ ] CPU 思考中に UI が固まらず、プレイヤー操作が二重送信されない。
- [ ] CPU の手が盤面、ログ、手番表示、履歴に同じ内容で反映される。

### 表示・演出・サウンド

- [ ] 駒、壁、セルハイライト、選択中スキル、ステータスアイコン、手番パネルが画面解像度を変えても崩れない。
- [ ] BGM/SE、背景効果、時間効果が再生・停止・切り替え時に重複や残留を起こさない。
- [ ] カタログや Prefab の未設定時に、致命的な NullReference ではなく検出しやすいログまたは安全な停止になる。
- [ ] 日本語テキスト、数値、ログがプレイヤーに理解できる粒度で表示される。

### 長時間・異常系

- [ ] 1 試合を最後まで複数回プレイし、メモリ増加、ログスパム、イベント購読漏れが目立たない。
- [ ] 連続リセット、勝敗直前のリセット、CPU 思考中のリセットで状態が壊れない。
- [ ] Editor とビルドで挙動差がない。

## 自動テストすべき観点

毎回の変更で退行しやすく、期待値をコードで明確に表現できる領域を自動化する。

### Domain / Rule

- [ ] 駒移動: 通常移動、壁による遮断、相手駒周辺、盤端、ゴール直前、無効座標。
- [ ] 壁配置: 方向別パターン、重複、交差、隣接、盤外、残数、両プレイヤーの経路保証。
- [ ] 経路探索: 最短距離、到達不可、非正方形/小型盤面、壁密度の高い盤面、境界条件。
- [ ] 勝敗判定: 1P/2P それぞれのゴール到達、同一ターン内イベント順序、終了後の状態固定。
- [ ] スキル合法性: 対象種別、対象プレイヤー、クールダウン、使用回数、ステータス制限、パラメータ不足。
- [ ] スキル効果: 移動、壁配置、壁除去、ステータス付与/解除、複合効果、イベント発行。
- [ ] ステータス適用: 再付与ポリシー、スタック、期限、クールダウン、ターン開始/終了時の発火順。
- [ ] 履歴・Memento: DeepCopy、Undo/Redo 相当、イベント履歴、参照共有がないこと。

### Application / Services

- [ ] `MatchCommandExecutor` が成功/失敗レスポンス、イベント発行、ターン進行を一貫して処理する。
- [ ] 各 `CommandHandler` が未対応コマンド、手番外プレイヤー、終了済み対戦を拒否する。
- [ ] `MatchInputCommandDispatcher` / `MatchInputReleaseValidator` が入力状態から正しいコマンドだけを生成する。
- [ ] `SkillSelectionController` が選択、解除、対象切替、拒否後復帰を正しく扱う。
- [ ] `StatusApplicator` / `StatusEffectApplicator` が定義値とイベントを同期させる。
- [ ] `MatchEventBus` が購読、解除、複数購読者、例外発生時の扱いを明確にする。

### Presentation / ViewModel

- [ ] Presenter がイベントから ViewModel を更新し、重複更新や古い状態の残留を起こさない。
- [ ] Board/Cell/Pawn/Wall の ViewModel が MatchState から正しく射影される。
- [ ] SkillButton/StatusPanel/TurnPanel の有効状態、表示文言、残数、クールダウン表示が正しい。
- [ ] View をモックまたは Spy に置き換えた Presenter 単体テストで、Unity オブジェクトに依存しない更新を確認する。

### CPU / Search

- [ ] Move generation が合法手のみを列挙し、重複手を出さない。
- [ ] 評価関数が距離、壁数、ゴール、危険状態などを期待どおりにスコア化する。
- [ ] Alpha-Beta が深さ、時間制限、キャンセル、同点手、詰み局面を扱う。
- [ ] SearchState の Apply/Undo を複数手連続で実行しても完全に元に戻る。
- [ ] 探索プロファイルがノード数、BFS 回数、経過時間を過不足なく計測する。

### GameFlow / UnityIntegration

- [ ] `GameDirector` が NewSession、Reset、エラー応答、ライフサイクルを処理する。
- [ ] `BoardGamePort` と `DebugPort` がドメインレスポンスを Unity 側へ変換する。
- [ ] カタログ類が必須 ID の有無、重複 ID、未設定参照を検出する。
- [ ] Sound/Background/TimeEffect サービスが再生、停止、同一 ID 再適用、未登録 ID を扱う。

## 自動テストで最低限足りていない部分

優先度 P0 は、ルール破綻や進行不能に直結するため最優先で追加する。

| 優先度 | 不足している自動テスト | 理由 | 推奨テスト種別 |
| --- | --- | --- | --- |
| P0 | `MovePawnValidator` の合法/違法移動 | 駒移動は最頻出操作で、現在のテストでは移動ルール全体が保護されていない。 | EditMode 単体 |
| P0 | `PlaceWallValidator` の壁配置合法性と経路保証 | 壁の重複・交差・経路封鎖はゲーム成立条件に直結する。 | EditMode 単体 |
| P0 | `MatchCommandExecutor` からターン進行・イベント発行までの成功/拒否パス | 個別の状態テストだけでは、実際のコマンド処理の接続ミスを検知しにくい。 | EditMode 結合 |
| P0 | `GoalResolver` / `MatchResultResolver` の勝敗判定 | ゴール到達と終了状態の退行はゲーム完了を壊す。 | EditMode 単体 |
| P0 | `SkillAvailabilityValidator` の拒否理由 | UI の有効/無効と実行可否の源泉であり、クールダウン・使用回数・対象不一致を守る必要がある。 | EditMode 単体 |
| P1 | `StatusApplicator` / `StatusEffectApplicator` の再付与・スタック・イベント | `PlayerState` 単体だけでは、定義からステータスを適用する経路が未保護。 | EditMode 単体/結合 |
| P1 | `Pathfinder` の壁あり盤面・到達不能盤面・最短距離 | 現在はオープン盤面中心で、壁配置ルールの基盤として不足している。 | EditMode 単体 |
| P1 | `SearchMoveGenerator` / CPU が合法手だけを返すこと | CPU が違法手を出すと対戦が破綻する。 | EditMode 単体 |
| P1 | `SearchState` の複数手 Apply/Undo と壁・駒混在シナリオ | 単発の巻き戻しだけでは探索中の状態破損を検知しにくい。 | EditMode 単体 |
| P1 | `MatchInputCommandDispatcher` と `SkillSelectionController` の入力状態遷移 | 手動操作で起きる選択解除・拒否後復帰のバグを自動で再現しやすい。 | EditMode 単体 |
| P2 | Presenter から ViewModel 更新までのイベント反映 | 表示退行を PlayMode に頼りすぎないため。 | EditMode + Spy |
| P2 | カタログの ID 重複・未設定参照検証 | ScriptableObject 設定ミスを早期に見つけるため。 | EditMode/PlayMode |
| P2 | `GameDirector` の NewSession/Reset/エラー応答 | 起動・リセットの結合不良を検知するため。 | PlayMode 結合 |
| P2 | Sound/Background/TimeEffect サービスの登録 ID と再適用 | 演出の残留や未登録 ID を検知するため。 | EditMode/PlayMode |

## 追加順の提案

1. P0 の Domain / Application テストを追加し、ゲームとして成立する最低限のルールを固定する。
2. P1 のステータス、探索、入力状態テストを追加し、CPU とスキルの退行を防ぐ。
3. P2 の Presenter、カタログ、UnityIntegration テストを追加し、手動確認の範囲を縮小する。
4. CI で Unity EditMode テストを常時実行し、PlayMode は主要シナリオを絞って定期実行する。
